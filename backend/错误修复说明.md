# 鲜花商城后端错误修复说明

## 修复的错误

### 1. 错误：No static resource api/category/list

**问题原因：**
- CategoryController 不存在
- 导致访问 `/api/category/list` 时出现 404 错误

**解决方案：**
✅ 创建了 `CategoryController.java`

文件位置：`src/main/java/com/flowershop/controller/CategoryController.java`

```java
@RestController
@RequestMapping("/api/category")
public class CategoryController {

    @Autowired
    private CategoryService categoryService;

    @GetMapping("/list")
    public Result list() {
        List<Category> categories = categoryService.getActiveCategories();
        return Result.success(categories);
    }
}
```

**测试方法：**
访问 `GET http://localhost:8088/api/category/list`
应该返回分类列表数据

### 2. 错误：io.jsonwebtoken.MalformedJwtException: Compact JWT strings may not contain whitespace

**问题原因：**
- 当 token 为空、错误、或包含空格时，系统直接调用 `JwtUtils.getUserId()` 抛出异常
- 没有异常处理机制，导致系统崩溃

**解决方案：**

#### 2.1 修复全局异常处理器
✅ 更新 `GlobalExceptionHandler.java`，添加 JWT 异常处理

```java
@ExceptionHandler(JwtException.class)
public Result<Void> handleJwtException(JwtException e) {
    log.warn("JWT验证失败: {}", e.getMessage());
    return Result.unauthorized();
}
```

#### 2.2 修复所有需要认证的 Controller

修复了以下 Controller，添加了健壮的 token 处理逻辑：

1. **CartController.java** - 购物车
   - 所有接口添加 token 空值和错误处理
   - 使用 try-catch 捕获 JWT 解析异常
   - 返回友好的错误提示而不是抛出异常

2. **UserController.java** - 用户
   - `getInfo` - 获取用户信息
   - `updateInfo` - 更新用户信息
   - `updatePassword` - 修改密码

3. **AddressController.java** - 地址
   - 所有增删改查接口
   - 添加统一的 `extractUserId()` 方法

4. **FavoriteController.java** - 收藏
   - 切换收藏状态
   - 收藏列表
   - 检查收藏状态

5. **OrderController.java** - 订单
   - 创建订单
   - 支付订单
   - 取消订单
   - 订单列表
   - 订单详情
   - 确认收货

6. **AiChatController.java** - AI 客服
   - 聊天接口
   - 获取历史记录
   - 清空历史记录

#### 2.3 添加购物车数量查询接口

✅ 在 `CartController.java` 添加了 `/api/cart/count` 接口
✅ 在 `CartService.java` 添加了 `getCartCount()` 方法

## Token 处理逻辑说明

所有修复后的 Controller 都采用了统一的 token 处理方式：

```java
private Long extractUserId(String token) {
    // 1. 检查 token 是否为空
    if (token == null || token.trim().isEmpty()) {
        return null;
    }

    try {
        // 2. 移除 "Bearer " 前缀
        String actualToken = token;
        if (token.startsWith("Bearer ")) {
            actualToken = token.substring(7);
        }

        // 3. 去除前后空格
        actualToken = actualToken.trim();
        if (actualToken.isEmpty()) {
            return null;
        }

        // 4. 解析 token 获取用户ID
        return JwtUtils.getUserId(actualToken);
    } catch (Exception e) {
        // 5. 任何异常都返回 null，不让系统崩溃
        return null;
    }
}
```

**错误处理方式：**

| Token 状态 | 处理方式 | 返回结果 |
|-----------|---------|---------|
| null | 返回 null | 返回 401 未登录 |
| 空字符串 "" | 去除空格后返回 null | 返回 401 未登录 |
| 只有空格 " " | 去除空格后返回 null | 返回 401 未登录 |
| 包含空格的 token | 去除空格后尝试解析 | 成功则继续，失败则返回 401 |
| 格式错误的 token | 捕获异常 | 返回 401 未登录 |
| 过期/无效的 token | 捕获异常 | 返回 401 未登录 |
| 有效的 token | 正常解析 | 返回用户ID，继续执行业务逻辑 |
